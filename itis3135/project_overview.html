Project Overview: ASA Connect

Web Site Name: ASA Connect Student: Vahid Zamen Date: November 2, 2025

Category: Student Group at UNCC (external-facing site)

Project Name

ASA Connect – UNCC Afghan Student Association

Project Overview

Purpose

Design and develop a responsive, accessible website for the UNCC Afghan Student Association (ASA) to centralize information, streamline communication, and encourage participation in events, cultural programs, and community projects. The application improves visibility, recruitment, and coordination without storing user data on our server (read‑only patterns with approved third‑party services/APIs only).

Intended Users

  • Current UNCC students interested in joining ASA activities.
  • Prospective students & families exploring campus cultural communities.
  • Alumni & community partners supporting mentorship, volunteering, and events.
  • Campus stakeholders (student org office, cultural centers) coordinating calendars & space.

Content Overview

  • About (mission, officers, constitution highlights)
  • Events (upcoming & past with media)
    • Read-only calendar embed (Google Calendar)
    • Event detail modals with photo gallery
  • Get Involved (join interest form, volunteer signups via third‑party forms)
    • Formspree/Google Forms submissions; client receives email; no on‑site storage
  • Resources (campus links, community orgs, scholarship list)
  • Gallery (interactive image grid, filters by tag)
  • Contact (officer emails via mailto, social links)

Page Design

Home (index.html)

  • Purpose: Snapshot of ASA; funnel to Events/Get Involved.
  • Audience: All visitors.
  • Content: Hero banner, 3 feature cards (Mission, Events, Join), latest news (AJAX), testimonials slider.
  • User Input: None directly.
  • Validation: N/A.
  • UI Elements: Buttons, nav links, jQuery UI Accordion for FAQs.
  • Actions: CTA buttons route to Events/Join; AJAX fetch for news from campus RSS.
  • Notes: Accessibility-first; lighthouse ≥ 95 targeted.

About (about.html)

  • Purpose: Mission, officers, constitution highlights.
  • Audience: Prospects, campus stakeholders.
  • Content: Officer cards, timeline (jQuery UI Datepicker stylings), downloadable PDF.
  • User Input: None.
  • Validation: N/A.
  • UI Elements: Tabs component to switch Mission/Officers/History.
  • Actions: Open mailto links; smooth-scroll within page.
  • Notes: Officer images optimized; lazy‑loading.

Events (events.html)

  • Purpose: Publicize upcoming events and archive past events.
  • Audience: Students, alumni, partners.
  • Content: Embedded Google Calendar (read‑only), event cards with images and tags.
  • User Input: Search/filter controls.
  • Validation: Client‑side only for filter text.
  • UI Elements: jQuery UI Autocomplete for tag search; modal dialog for event details.
  • Actions: Filter events by tag/date; open event modal; Add‑to‑Calendar deep link.
  • Notes: All data read-only via embed or static JSON.

Get Involved (get-involved.html)

  • Purpose: Capture interest/volunteer preferences via third‑party forms.
  • Audience: Prospective members, volunteers.
  • Content: Join form (Formspree/Google Forms), role checkboxes, availability, interests.
  • User Input: Yes – name, email, role, interest areas, consent checkboxes.
  • Validation: Required fields, email pattern, character limits, terms consent.
  • UI Elements: jQuery UI Slider for availability hours; Progressbar during submit.
  • Actions: Submit to third‑party endpoint; display success/fail message (no on‑site storage).
  • Notes: reCAPTCHA or anti‑bot honeypot where supported by provider.

Resources (resources.html)

  • Purpose: Curate links to campus services, community groups, scholarships.
  • Audience: Students & families.
  • Content: Filterable list with tags (Aid, Mentorship, Cultural Centers).
  • User Input: Tag filters, search box.
  • Validation: N/A.
  • UI Elements: jQuery UI Accordion for categories.
  • Actions: Client-side filtering; external links open in new tab with rel attributes.
  • Notes: Provide short descriptions and accessibility labels.

Gallery (gallery.html)

  • Purpose: Showcase event photos to boost engagement.
  • Audience: All visitors.
  • Content: Masonry grid with lazy‑loaded thumbnails and lightbox.
  • User Input: Filter by tag/year.
  • Validation: N/A.
  • UI Elements: jQuery lightbox plugin; jQuery UI Checkboxradio for filters.
  • Actions: Open modal viewer; keyboard navigation; ARIA roles for accessibility.
  • Notes: Images compressed; captions include alt text credits.

Contact (contact.html)

  • Purpose: Provide safe contact paths without on‑site data storage.
  • Audience: Anyone seeking contact.
  • Content: Officer roles with mailto links; social media; embedded map.
  • User Input: Optional redirect to third‑party contact form.
  • Validation: Basic email pattern if using third‑party embed.
  • UI Elements: Buttons, links, map iframe.
  • Actions: Open mail client; map interactions; link to Get Involved form.
  • Notes: Avoid exposing personal numbers; use role accounts where possible.

Dynamic Functionality (Read‑Only + Client‑Side)

  • Interactive gallery with filtering and lightbox (page: Gallery). Example inspiration: PhotoSwipe.
  • FAQ accordion (page: Home/Resources) using jQuery UI Accordion. Inspiration: jQuery UI Accordion.
  • Events search/autocomplete (page: Events) using jQuery UI Autocomplete sourced from a static events.json. Inspiration: jQuery UI Autocomplete.
  • AJAX campus news (page: Home) client‑side fetch of a public RSS/JSON feed transformed in‑browser for a read‑only news strip. Inspiration: Fetch API.
  • Join/Volunteer forms (page: Get Involved) via Formspree or Google Forms; display success message with jQuery UI Dialog.
  • Map embed (page: Contact) via Google Maps iframe (pure read‑only).

Global Components & Standards

  • Navigation: Single‑click access to each top‑level page; submenus permitted if more pages are added.
  • Header/Footer: Consistent across pages; footer includes HTML & CSS validator links.
  • Accessibility: WCAG 2.2 AA targets; semantic landmarks; focus states; color contrast ≥ 4.5:1.
  • Performance: Image optimization, code splitting, defer scripts, preconnect for embeds.
  • No on‑site data storage: All interactivity is read‑only or delegated to approved third‑party forms/APIs.
