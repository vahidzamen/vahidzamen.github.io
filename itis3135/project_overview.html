Project Overview — “49ers Car Culture Club” (C3)

Client Category (Step 01): Category 03 – student group at UNCC
Project Category (Step 02): Student group site that encourages participation & communication

1) Application Purpose

The 49ers Car Culture Club (C3) needs a simple, fast site to organize events (meets, cruises, tech nights), recruit new members, and share media from the community. The web application improves visibility, centralizes announcements, and offers light interactivity (photo gallery, interest form, event RSVP to a Google Sheet, and read-only pulls from public APIs).

2) Intended Users

  • Prospective members: Students curious about joining (want basics, how to join, upcoming events).
  • Active members: Need dates, locations, and quick RSVP links.
  • Officers: Need a clear place to post schedules and rules (content provided to developer).
  • Partners/Sponsors: View past events, contact officers for collaborations.

3) Content Overview

  • About/mission: Who we are, safety guidelines, officer bios.
  • Events: Calendar of meets & cruises, event detail pages.
  • Gallery: Curated photos from members (interactive lightbox).
  • Join/Contact: Interest form (validations) + map to meet locations.
  • Resources: Links to campus policies, local tracks, detailing guides.

8) Page Design Details

Home (index.html)
  • Purpose: Welcome, value prop, highlight next event.
  • Audience: Everyone (prospects, members, partners).
  • Content: Hero, two-sentence mission, “Next Event” card, 3-image gallery preview, “Join” CTA.
  • Inputs: None.
  • Controls: Buttons/links to Events & Join pages.
  • Actions: “Next Event” card opens detail (modal or link).
  • Notes: jQuery-UI Accordion for FAQs at bottom.
About (about.html)
  • Purpose: Explain mission, expectations, officer bios.
  • Audience: Prospective members, campus partners.
  • Content: Mission paragraph, safety checklist, officer cards.
  • Inputs: None.
  • Controls: “Contact an Officer” link to Join page.
  • Actions: Expand safety rules (Accordion).
Events (events.html)
  • Purpose: Central list of upcoming/past events.
  • Audience: Members & prospects.
  • Content: Event cards (title, date, time, location, brief).
  • Inputs: None (read-only).
  • Controls: “Details” button opens modal; “RSVP” opens Google Form.
  • Validations: N/A (handled by Google Form if used).
  • Actions: AJAX (read-only) fetch to load a public JSON of upcoming events (hosted on GitHub Pages or a static JSON file) and render cards.
Gallery (gallery.html)
  • Purpose: Showcase photos from meets.
  • Audience: Everyone.
  • Content: Responsive grid of thumbnails.
  • Inputs: None (no uploads; read-only per course rules).
  • Controls: Click thumbnail → lightbox viewer with prev/next.
  • Actions: jQuery plugin for lightbox; keyboard navigation.
Join & Contact (join.html)
  • Purpose: Collect interest & questions.
  • Audience: Prospective members/partners.
  • Content: Form fields (name, UNCC email, grad year, car make/model [optional], interests, message), Google Map embed (meet spot).
  • Inputs: Form with client-side validations (required fields, email pattern, character limits).
  • Controls: Submit button + Reset.
  • Actions: On submit, validate → either:
    • Open a prefilled mailto: to the officers, or
    • POST to a Google Form endpoint (if allowed) or redirect to Google Form (no custom CRUD).
  • Notes: Confirmation message displayed on success.
Resources (resources.html) — optional
  • Purpose: Link out to campus policies, track days, safety guides.
  • Actions: External links open in new tabs.

9) Planned Interactivity (JavaScript/jQuery/AJAX)

  • jQuery-UI Accordion for FAQs/Safety on Home/About.
  • Gallery Lightbox (jQuery plugin) with keyboard nav.
  • Events Loader (AJAX, read-only): Fetch a static public JSON (hosted file) to render upcoming events—no user writes.
  • Form Validations: HTML5 + JS (required, pattern, length). Optional integration with Google Forms for submissions.
  • Map Embed: Simple Google Maps iframe to meet location.

10) Example Inspiration (interaction references)

  • University club landing pages with clear CTAs & events lists.
  • Accessible image galleries with keyboard-friendly lightboxes.
  • Read-only event feeds generated from static JSON.
