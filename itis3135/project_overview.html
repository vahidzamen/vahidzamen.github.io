Project Overview — 49ers Car Culture Club (C3)

Client & Project Category

Client Category (Step 01): Student group at UNCC
Project Category (Step 02): Group site that encourages participation & communication

1) Application Purpose

The 49ers Car Culture Club (C3) site organizes events (meets, cruises, tech nights), recruits members, and shares media. It improves visibility, centralizes announcements, and offers light interactivity (gallery, FAQs, event highlights, and a read-only events list).

2) Intended Users

  • Prospective members: basics, how to join, upcoming events
  • Active members: dates, locations, quick RSVP links
  • Officers: publish schedules & rules (content provided)
  • Partners/Sponsors: view past events & contact

3) Content Overview

  • About/mission, officer bios, safety guidelines
  • Events list and highlights
  • Gallery with accessible lightbox
  • Join/Contact interest form + campus map
  • Resources (optional): policies & useful links

7) Required Pages & Navigation

  • Home (index.html) — overview, next event highlight, Join CTA
  • About (about.html) — mission, officer bios, safety rules
  • Events (events.html) — list of upcoming/past events (read-only)
  • Gallery (gallery.html) — photo grid with lightbox
  • Join & Contact (join.html) — form with validation + map
  • Optional: Resources (resources.html) — policies & links

8) Page Design Details

Home (index.html)
  • Purpose: Welcome users and show next event quickly
  • Audience: Everyone
  • Content: Hero, mission, next event card, gallery preview, CTA
  • Actions: Details button → Events
About (about.html)
  • Purpose: Explain mission, values, officers
  • Actions: Expandable FAQ or safety rules
Events (events.html)
  • Purpose: Show all upcoming/past meets
  • Actions: Read-only AJAX event loading from static JSON
Gallery (gallery.html)
  • Purpose: Showcase photos of past events
  • Actions: Lightbox viewer
Join & Contact (join.html)
  • Purpose: Collect interest + questions
  • Validations: Required fields + email pattern
  • Actions: Mailto or Google Form
Resources (resources.html)
  • Purpose: External links to policies & tools

9) Planned Interactivity

  • jQuery UI Accordion (FAQ / Safety)

  • Events loaded from static JSON (read-only)
